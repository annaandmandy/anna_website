# 對話格式說明 - Dialogue Format Guide

## 📝 對話結構

每段遊戲文本現在都是**對話陣列**，包含多個對話物件：

```json
{
  "場景名稱": {
    "dialogue": [
      {
        "speaker": "說話者",
        "text": "對話內容",
        "emotion": "情緒描述（可選）"
      }
    ]
  }
}
```

## 🎭 說話者類型

### 主要角色
| 說話者 | 用途 | 範例 |
|--------|------|------|
| **我** | 主角的對話 | "我想去冷泉。" |
| **我** (with 括號) | 主角的內心想法 | "（這裡有點奇怪...）" |
| **旁白** | 場景描述、動作描述 | "我推開通往溫泉的門。" |
| **系統** | 遊戲系統訊息 | "【理智值 -15】" |

### NPC 角色
| 說話者 | 角色 | 特點 |
|--------|------|------|
| **母親** | 主角的母親 | 推薦溫泉、循環對話 |
| **櫃台人員** | 溫泉接待員 | 第一個遇到的工作人員 |
| **工作人員** | 溫泉員工 | 可能觸發特殊事件 |
| **訪客A/B** | 其他溫泉訪客 | 看似正常但... |
| **訪客們** | 多位訪客同時 | 集體反應 |
| **它們** | 未知生物 | 恐怖元素 |
| **未知生物** | 水下的東西 | 最終揭曉 |

## 💬 對話格式範例

### 1. 基本對話
```json
{
  "speaker": "櫃台人員",
  "text": "歡迎光臨！第一次來嗎？"
}
```

### 2. 帶情緒的對話
```json
{
  "speaker": "工作人員",
  "text": "請務必跟我來。",
  "emotion": "笑容越來越詭異"
}
```

### 3. 內心想法（用括號）
```json
{
  "speaker": "我",
  "text": "（為什麼鏡子都被遮起來了？）"
}
```

### 4. 旁白/場景描述
```json
{
  "speaker": "旁白",
  "text": "冷水讓我瞬間清醒。"
}
```

### 5. 系統訊息
```json
{
  "speaker": "系統",
  "text": "【理智值 -15】"
}
```

### 6. 恐怖氛圍營造
```json
{
  "speaker": "訪客們",
  "text": "加入我們...加入我們...",
  "emotion": "低語聲從四面八方傳來"
}
```

## ✍️ 寫作技巧

### 1. 節奏控制
- 短對話 = 緊張急促
- 長對話 = 平靜敘事
- 「......」= 沉默、氛圍

### 2. 恐怖元素
```json
// ❌ 直接描述
"水裡有鯊魚很可怕"

// ✅ 漸進式恐怖
{
  "speaker": "我",
  "text": "（水面下...有東西在動...）"
},
{
  "speaker": "旁白", 
  "text": "那不是人類的輪廓。"
},
{
  "speaker": "我",
  "text": "...魚鰭？！"
}
```

### 3. 內心 vs 外在
```json
// 外在對話（說給別人聽）
{
  "speaker": "我",
  "text": "我很好，謝謝。"
}

// 內心想法（真實感受）
{
  "speaker": "我",
  "text": "（必須逃離這裡...）"
}
```

### 4. 情緒標記使用
```json
// 增強氛圍
{
  "speaker": "工作人員",
  "text": "歡迎光臨。",
  "emotion": "微笑但眼神空洞"
}

// 動作描述
{
  "speaker": "訪客A",
  "text": "......",
  "emotion": "被打倒在地，但面無表情"
}
```

## 🎯 不同場景的對話風格

### 日常場景（家中）
- 溫馨、正常的對話
- 為反差做鋪墊

```json
{
  "speaker": "母親",
  "text": "你最近看起來壓力很大。不如去溫泉放鬆一下？"
}
```

### 緊張場景（發現異常）
- 短句
- 破折號、驚嘆號
- 內心獨白

```json
{
  "speaker": "我",
  "text": "那是...什麼...？！"
}
```

### 恐怖場景（被追捕）
- 快速對話
- 多個角色同時
- 重複語句

```json
{
  "speaker": "工作人員們",
  "text": "不穩定的訪客。啟動處置程序。"
}
```

### 結局場景
- 較長的旁白
- 總結性對話
- 餘韻留白

```json
{
  "speaker": "旁白",
  "text": "從此，池中多了一個影子。等待著下一位訪客..."
}
```

## 📋 快速檢查清單

寫對話時確認：
- [ ] 每個對話都有 `speaker` 和 `text`
- [ ] 使用 `()` 標記內心想法
- [ ] 使用 `【】` 標記系統訊息
- [ ] 情緒標記用於增強氛圍
- [ ] 對話節奏符合場景氛圍
- [ ] 避免純粹的敘述，要有互動感

## 🎨 範本庫

### 工作人員引導
```json
{
  "speaker": "工作人員",
  "text": "請跟我來。有個更適合您的池子。",
  "emotion": "笑容僵硬"
}
```

### 恐怖發現
```json
{
  "speaker": "我",
  "text": "等等...那是...！"
},
{
  "speaker": "旁白",
  "text": "我看見了不該看的東西。"
}
```

### 理智降低
```json
{
  "speaker": "我",
  "text": "（頭好暈...這些人的臉...）"
},
{
  "speaker": "旁白",
  "text": "周圍的聲音開始扭曲。"
}
```

### 集體異常
```json
{
  "speaker": "訪客們",
  "text": "......",
  "emotion": "全部同時轉頭看向我"
}
```

## 💡 現在開始編寫你的恐怖對話！

所有 JSON 文件都已改為對話格式，你可以：
1. 直接編輯 `locations.json`、`events.json`、`endings.json`
2. 添加更多對話分支
3. 創造新的 NPC 角色
4. 設計更多恐怖橋段

記住：**好的對話不是告訴玩家發生了什麼，而是讓玩家感受到正在發生什麼！** 🎭👻
